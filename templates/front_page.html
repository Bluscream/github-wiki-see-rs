GitHub Wiki SEE (decomissioned)

GitHub Wiki Search Engine Enablement previously allowed GitHub Wikis to be indexed by search engines. There was no robots.txt on this site preventing indexing of wiki content and wiki content proxied from GitHub and served from this domain will be crawled and indexed.

Previously, GitHub prevented crawling of repository Wiki pages. This means GitHub Wikis are invisible to search engines such as Google and DuckDuckGo and were an a very unpleasant surprise if you're looking to share searchable knowledge or put a FAQ on a Wiki!

GHWSEE had simply proxied the rendered contents of the pages and has a link at the top to visit the page on GitHub for normal usage and a link at the bottom explaining what GHWSEE is. The content of this service was not meant to be read by users and was meant for crawlers. It was intended to behave like those StackOverflow content mirroring sites but with far less ad-farming.

For anti-abuse reasons, all links rendered in the service that are going out of GitHub are tagged with rel="nofollow ugc" as to not affect search engine rankings or to promote mass vandalism of GitHub Wikis. This was likely the original reason for the indexing blocking.

Usage

  1. Used a search engine! That was the intended interface to the content "on" this site.

If you were an editor, you edited your page periodically and the sitemap generator will eventually pick it up and notify search engines! Some stuff ranked pretty high but it wasn't a slam dunk to be "number 1".

Note that it had taken quite some time for content to be picked up and there is no guarantee it'll be picked up.
